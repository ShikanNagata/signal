# 『俺の買いシグナル』3×3 マトリックス 自動生成ツール

過去の株価データから「エントリー時のシグナル状態 × 保有日数」の期待値マトリックスを自動生成します。
**Cowork でも、普通のターミナルでも動きます。**

---

## 📦 フォルダの中身

```
signal-matrix/
├── run.py          ← メインプログラム（これを実行するだけ）
├── config.json     ← 設定ファイル（銘柄・期間などを編集）
├── README.md       ← この説明書
├── data/           ← （任意）yfinanceが使えない時のCSV置き場
└── results/        ← 結果が日付つきで自動保存される
```

---

## 🚀 Cowork での使い方（一番かんたん）

### 初回セットアップ（1回だけ）

1. **このフォルダ（signal-matrix）をPCの好きな場所に置く**
   - 例: `書類/signal-matrix/`

2. **Claude Desktop アプリを開いて Cowork モードにする**

3. **「フォルダで作業」→ signal-matrix フォルダを選ぶ**

4. **Coworkにこう頼む:**
   ```
   このフォルダで run.py を実行して。
   必要なライブラリ（yfinance, pandas, numpy）が無ければ先に入れて。
   ```

これだけ。Coworkが自動で：
- 足りないライブラリをインストール
- config.json の銘柄の株価を取得
- マトリックスを計算
- `results/` にHTMLとJSONを保存

### 2回目以降（毎回の更新）

Coworkにこう言うだけ：
```
最新データでマトリックスを更新して。前回との変化も教えて。
```

## 💻 ターミナルでの使い方（Coworkを使わない場合）

```bash
# 初回だけ：ライブラリをインストール
pip install yfinance pandas numpy

# フォルダに移動して実行
cd signal-matrix
python run.py
```

実行が終わると `results/matrix_YYYY-MM-DD.html` が生成されます。
ブラウザで開けばマトリックスが見られます。

## ⚙️ 設定の変え方（config.json）

`config.json` をテキストエディタで開いて編集します。

### 銘柄を変える
```json
"tickers": ["WULF", "AMD", "MRVL", "CRCL"]
```
ここを書き換えるだけ。増やしてもOK：
```json
"tickers": ["WULF", "AMD", "MRVL", "IONQ", "NBIS", "MSTR"]
```

### 保有株を登録する（含み損益も表示されるようになる）
```json
"my_holdings": {
  "WULF": {"qty": 140, "avg_cost_usd": 15.04},
  "AMD":  {"qty": 17,  "avg_cost_usd": 296.72}
}
```

### 過熱の基準を変える
```json
"rsi_overheated": 70
```
70を80にすれば「もっと過熱しないと過熱判定しない」になります。

### 保有日数の刻みを変える
```json
"hold_days": [5, 10, 20, 40, 80]
```

## 🔧 yfinanceが使えない時（CSVフォールバック）

会社のPCなどで yfinance がブロックされている場合、手動CSVでも動きます。

1. Yahoo FinanceやGoogle Financeで各銘柄の株価CSVをダウンロード
2. `data/` フォルダに `WULF.csv` `AMD.csv` のように銘柄名で保存
3. CSVの列は `Date, Open, High, Low, Close, Volume` の順
4. `python run.py` を実行 → 自動で data/ のCSVを読みます

### Google Sheetsで取る場合
```
=GOOGLEFINANCE("NASDAQ:WULF","all","2024-11-01","2026-06-21")
```
を入力して、CSVでダウンロード → `data/WULF.csv` にリネーム。

## 📊 マトリックスの読み方

- **縦軸**: エントリー時のシグナル状態
  - `3/3 健全` = 5/50/200日MA全部上 + RSI 70未満（強いが過熱でない）
  - `3/3 過熱` = 全部MA上だがRSI 70以上（過熱、反落注意）
  - `2/3 警戒` = MAを2本上回る
  - `1/3 弱い` = MAを1本だけ上回る（押し目、逆張りチャンス）
  - `0/3 底値` = MA全部下（売られすぎ、底値圏）

- **横軸**: 何日保有したか（5/10/20/40/80日）

- **セルの色**: 緑=期待値プラス、赤=マイナス。濃いほど強い

- **NOWマーカー**: 各銘柄が今どのシグナル状態にいるか

### 使い方の例
「今WULFが『1/3 弱い』にいて、そのセルが緑（+30%）なら、過去データ上は買い場」
「AMDが『3/3 過熱』で、40日保有のセルが赤なら、長期保有は危険」

## ❓ よくある質問

**Q: データはどれくらいの期間必要？**
A: MA200を使うので最低300日、できれば1年半（550日）以上。config.jsonの`history_period`で調整。

**Q: 新しい銘柄を足したい**
A: config.jsonの`tickers`に追加するだけ。IPO直後でMA200が無い銘柄は自動で5/20/50MAに切り替わります。

**Q: 結果はどこに保存される？**
A: `results/matrix_YYYY-MM-DD.html`（ダッシュボード）と`.json`（生データ）。日付つきなので過去と比較できます。

**Q: バックテストの数字は信用できる？**
A: あくまで過去の統計です。2024年後半以降の上昇相場が主体のデータなので、下落相場での再現性は限定的。参考指標として使ってください。
